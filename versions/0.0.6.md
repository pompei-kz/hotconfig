# Version 0.0.6

Released: 2026-05-24

## Summary

Version 0.0.6 moves factory construction to `HotConfigFactoryBuilder`, splits dynamic runtime inputs into focused `Clock` and `EnvSrc`
interfaces, and adds persistent parameter error metadata across the built-in storage backends.

## Added

- `HotConfigFactory.builder()` as the public factory construction entry point.
- `HotConfigFactoryBuilder` options for:
  - `tunnel`
  - `clock`
  - `envSrc`
  - `extension`
  - `revisionCheckTimeoutMs`
- `Clock` for revision-check timing.
- `EnvSrc` for `$ENV{name}` value substitution.
- `ConfParam.error` persistence in file, JDBC, and etcd backends.
- `ConfigTunnelJdbcBuilder.colError(...)`, defaulting to `error`.
- PostgreSQL and MariaDB table creation support for the `error TEXT` column.
- Public API Javadocs for `HotConfigFactory`, `HotConfigFactoryBuilder`, and `ConfigTunnelFileBuilder`.
- Test utilities `ClockFake` and `EnvSrcFake`.

## Changed

- File and etcd tunnels write parameter errors immediately after the parameter line:

  ```text
  name=value
  #ERROR error text
  ```

- File and etcd error text uses the same escaping rules as values:
  - newline is written as `\n`
  - backslash is written as `\\`
  - escaped text is restored when read
- JDBC stores parameter errors in the configured `colError` text column.
- README examples now use `HotConfigFactory.builder()`.
- README dependency examples now use version `0.0.6`.

## Removed

- `HotConfigFactoryParams`.
- `HotConfigFactoryParamsBuilder`.
- `DynamicParams`.
- `DynamicParamsFake`.

## Migration Notes

- Replace direct factory construction and params usage with the builder:

  ```java
  HotConfigFactory factory = HotConfigFactory.builder()
    .tunnel(tunnel)
    .extension(".hotconf")
    .build();
  ```

- If you provided dynamic time or environment behavior in tests, pass separate implementations:

  ```java
  HotConfigFactory factory = HotConfigFactory.builder()
    .tunnel(tunnel)
    .clock(clock)
    .envSrc(envSrc)
    .build();
  ```

- If you already have JDBC config tables, add the error column manually. With default settings:

  ```sql
  error TEXT
  ```
