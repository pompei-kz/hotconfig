# Version 0.0.7

Released: 2026-05-25

## Summary

Version 0.0.7 refines parameter error storage for file and etcd tunnels. Multiline errors are now stored as repeated prefixed lines,
and error text is kept raw per line instead of using value-style escaping.

## Added

- `ConfigTunnelEtcdBuilder` and `ConfigTunnelEtcd.builder()` as the public etcd tunnel construction API.
- `ConfigTunnelJdbcBuilder` and `ConfigTunnelJdbc.builder()` as the public JDBC tunnel construction API.
- `ConfigTunnelEtcdBuilder.errorPrefix(...)` for configuring the text after the mandatory leading `#` on etcd tunnel error lines.
- `ConfigTunnelFileBuilder.errorPrefix(...)` for configuring the text after the mandatory leading `#` on file tunnel error lines.
- Tests for `ConfigTunnelMem` covering:
  - missing reads
  - config write/read behavior
  - defensive copies on write and read
  - modification marker increments
  - notice line storage
  - local path isolation

## Changed

- `ConfigTunnelFile` now stores the parameter error prefix in `ConfigTunnelFile.Def.errorPrefix`.
- `ConfigTunnelEtcd` now stores the parameter error prefix in `ConfigTunnelEtcd.Def.errorPrefix`.
- `ConfigTunnelJdbc` now stores table and column names in `ConfigTunnelJdbc.Def`, configured by `ConfigTunnelJdbcBuilder`.
- The default file and etcd tunnel error prefix is `ERROR `, producing the existing `#ERROR ` format by default.
- `ConfigTunnelFile` always prepends `#` to parameter error lines, so the comment marker is not part of the configurable prefix.
- `ConfigTunnelEtcd` always prepends `#` to parameter error lines, so the comment marker is not part of the configurable prefix.
- File and etcd tunnels now write multiline parameter errors as repeated prefixed lines:

  ```text
  name=value
  #ERROR first line
  #ERROR second line
  ```

- File and etcd tunnels read all consecutive error-prefixed lines after a parameter and join them back with real newline characters.
- Backslashes in file and etcd parameter error text are no longer escaped on write or unescaped on read.
- Regular parameter values keep the existing escaping rules.
- File and etcd `writeNoticeLines` delete the notice file or key when passed an empty list.
- Etcd key access logic now lives directly in `ConfigTunnelEtcd`; the internal `EtcdStorage` abstraction was removed.
- README examples now use version `0.0.7`.

## Removed

- `ConfTunnelEtcdDef`; configure etcd tunnels through `ConfigTunnelEtcd.builder()` instead.
- `ConfigTunnelJdbcDef`; configure JDBC tunnels through `ConfigTunnelJdbc.builder()` instead.

## Migration Notes

- If code configures a custom file tunnel error prefix, pass it without the leading `#`:

  ```java
  ConfigTunnelFile tunnel = ConfigTunnelFile.builder()
    .baseDir(baseDir)
    .errorPrefix("ERR ")
    .build();
  ```

  This writes errors as `#ERR ...`.

- Construct etcd tunnels with the builder:

  ```java
  ConfigTunnelEtcd tunnel = ConfigTunnelEtcd.builder()
    .client(client)
    .keyPrefix("/kz-pompei-conf-etcd/")
    .build();
  ```

- Construct JDBC tunnels with the builder:

  ```java
  ConfigTunnelJdbc tunnel = ConfigTunnelJdbc.builder()
    .connectionGet(() -> dataSource.getConnection())
    .tableName("hotconfig")
    .build();
  ```

- Existing file or etcd data with old single-line escaped errors can still be read as one error line, but it will not be expanded from
  literal `\n` text by the new error-line parser. Rewrite affected configs to the repeated-line format when multiline error text matters.
