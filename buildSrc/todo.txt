The kz-pompei-hotconfig-core module has publishing configured in its build.gradle file.
But writing so much code in each module isn't exactly chic.
It all needs to be moved to a separate KzPompeiMavenPublication plugin.
Create the plugin in the buildSrc directory.
In the build.gradle file, I specified which parameters need to be exposed via the plugin configuration so that only these parameters need to be specified in modules.

kzPompeiMavenPublication {

  pom {
    name = 'kz-pompei-hotconfig-core'
    description = 'Core part of hot configuration library'
    url = 'https://pompei.kz/programs/kz-pompei-hotconfig'
  }

  developers {
    developer {
      id = 'pompei'
      name = 'Evgenij Kolpakov'
      email = 'pompei@mail.ru'
    }
  }

  scm {
    connection = 'scm:git:https://github.com/pompei-kz/hotconfig.git'
    developerConnection = 'scm:git:ssh://git@github.com:pompei-kz/hotconfig.git'
    url = 'https://github.com/pompei-kz/hotconfig'
  }
}
