Two more methods have been added to the ConfigTunnel interface, and they need to be implemented in all implementations.
A builder has already been added to ConfigTunnelFile, and the old method of creating this object no longer works.
The corresponding tests need to be adjusted and new ones written for the new methods. The JavaDoc explains what needs to be done.

In the JDBC implementation, we need to expand the table columns by adding a TEXT field
to store the \n-delimited string data. There's no need to worry about the database table not having this field—let users do it manually.
It's inconvenient for them, but there are only... zero of them, so it's a no-brainer.

For this field, use a string related to the config, not its parameter. For parameters, leave this field null.

The etcd implementation needs to be implemented in a similar way to ConfigTunnelFile.

For all implementations, write tests for these methods.
